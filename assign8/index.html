About this app

Instructions

We have implemented two interfaces for this assignment where we either use one key or two keys. You can select your preferred interface on the home page. Also, after selecting a interfacce, you can change it by clicking the back button which redirects you to home page and there you can choose your interface. The instructions for using the interfaces are given below:

Instruction for one key/switch interface

We have implemented row-column scanning for one key interface. In this interface, a scanner cycles through the rows first, you press spacebar to select a particular row, then the scanner cycles through the row's values where you again press spacebar to select the value.

Instruction for two key/switch interface

For two key interface, use spacebar to go to next value and press return(enter) key to select the value.

Discussion

From the above experiments and results, it is seen that we are able to complete the task faster using two key interface than one key interface. We also see that the average number of errors is far less when using two key interface. This is supported by both test scenarios. This might be due to the fact that in one key interface, user cant dynamically control the flow, whereas in two key interface user has control of when to go to next value and when not; this contributes to a lot to making less error when using two key interface. When comparing test scenarios (hands v/s feet), it is seen that the average time taken to complete the task using hands is less than using feet and we tend to make significantly less error when using hands. This might be due to the experience and dexterity we have with our hands which is lacking when using feet. Also, comparing different speeds for one-key interface, we see that by increasing speed (reducing interval), we are able to complete the task faster. However, we also made more error.
